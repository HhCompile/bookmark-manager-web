# 项目初始化追踪器

## 触发规则

当调用此技能时，模型应遵循以下规则：

### 1. 执行规则
- **项目特定性**：仅针对当前工作目录中的项目进行操作，不影响其他项目
- **数据隔离**：所有数据存储在当前项目的 `.trae/skills/init-tracker/` 目录中
- **非破坏性**：执行操作时不修改项目的核心文件，仅记录和分析
- **实时更新**：每次操作后及时更新状态信息和相关记录

### 2. 技术分析规则
- **自动检测**：自动分析项目的技术栈、UI框架和包管理器
- **优先使用**：优先推荐使用项目已有的技术和库
- **插件优先**：如需新功能，优先推荐成熟的插件而非从零封装
- **兼容性评估**：评估推荐技术与现有技术栈的兼容性

### 3. 输出规则
- **结构化输出**：所有输出应采用清晰的结构化格式
- **详细记录**：记录执行过程的每一步，包括命令、结果和时间
- **问题优先级**：对发现的问题进行优先级排序
- **具体建议**：提供具体、可操作的技术使用建议

### 4. 通用适配规则
- **跨技术栈**：支持多种前端/后端技术栈
- **跨包管理器**：适配 npm、yarn、pnpm 等不同包管理器
- **跨框架**：兼容 React、Vue、Angular 等不同前端框架
- **环境无关**：适用于不同的开发环境和操作系统

## 功能描述

该技能用于：
1. **初始化状态检测**：检查项目是否已经完成初始化，识别未初始化的组件
2. **执行过程记录**：创建详细的文档记录项目初始化的每一步执行过程
3. **问题与建议管理**：跟踪待处理的问题和提供改进建议
4. **实时进程更新**：根据项目的实时进展及时更新状态信息
5. **技术栈分析**：自动判断项目使用的技术栈、UI框架和包管理器
6. **技术优先使用**：优先使用项目中已有的技术，如需新功能优先使用插件而非从零封装
7. **任务提醒**：给后续执行的任务或技能提供提醒和具体要求

## 适用场景

- 新项目启动时，需要系统化地跟踪初始化过程
- 接管现有项目时，需要快速了解项目的初始化状态
- 团队协作中，需要共享项目初始化的进展和问题
- 项目维护时，需要回顾初始化过程以了解项目架构

## 使用方法

### 1. 初始化状态检测

运行状态检测命令，系统会自动分析项目结构，检查以下内容：
- 依赖项是否已安装
- 配置文件是否完整
- 必要的目录结构是否存在
- 基础功能是否可正常运行

### 2. 技术栈分析

系统会自动检测并分析项目的技术栈：

#### 技术栈识别
- **前端框架**：检测是否使用 React、Vue、Angular 等框架
- **后端技术**：识别 Node.js、Python、Java 等后端技术
- **数据库**：检测项目使用的数据库类型
- **构建工具**：识别 Webpack、Vite、Rollup 等构建工具

#### UI框架检测
- 识别项目使用的 UI 组件库（如 Ant Design、Element Plus、Material UI 等）
- 分析组件的使用模式和定制化程度

#### 包管理器识别
- 检测项目使用的包管理器（npm、yarn、pnpm 等）
- 分析依赖项的管理方式和版本控制策略

#### 技术使用建议
- 根据检测结果，提供符合项目技术栈的最佳实践建议
- 推荐与现有技术栈兼容的插件和工具

### 3. 执行过程记录

系统会创建结构化的文档，记录：
- 初始化的时间和执行者
- 每一步的执行命令和结果
- 遇到的问题和解决方案
- 完成的里程碑和交付物

### 4. 问题与建议管理

系统会维护两个清单：
- **待处理问题**：记录初始化过程中发现的问题，包括优先级和责任人
- **改进建议**：提供优化项目结构、配置或流程的建议

### 5. 实时进程更新

- 支持手动更新状态
- 可与 CI/CD 流程集成，自动更新状态
- 提供状态变更的历史记录

### 6. 技术优先使用原则

系统会遵循以下技术使用原则：

#### 优先使用现有技术
- 分析项目中已有的技术栈和依赖项
- 优先使用项目已集成的技术和库
- 保持技术栈的一致性和统一性

#### 插件优先策略
- 如需添加新功能，优先查找并使用成熟的插件
- 评估插件的兼容性、维护状态和社区支持
- 提供插件集成的具体步骤和配置建议

#### 从零封装的条件
- 当现有插件无法满足需求时，才考虑从零封装
- 评估封装的成本和收益
- 提供封装方案的详细设计和实现建议

### 7. 任务提醒

系统会为后续执行的任务或技能提供详细的提醒：

#### 任务上下文提醒
- 提供项目当前状态的详细上下文
- 总结已完成的步骤和待处理的事项
- 突出显示需要特别关注的问题和风险

#### 技术使用要求
- 明确后续任务应使用的技术栈和工具
- 提供技术使用的具体规范和最佳实践
- 提醒避免技术栈的冲突和冗余

#### 执行步骤建议
- 提供后续任务的推荐执行顺序
- 建议每个步骤的具体操作和验证方法
- 预估每个步骤的时间和资源需求

#### 质量保证要求
- 明确代码质量和测试覆盖率的要求
- 提供代码审查的重点和检查清单
- 建议性能优化和安全考虑的要点

## 集成方式

### 命令行集成

```bash
# 检查初始化状态
trae skill init-tracker check

# 更新初始化状态
trae skill init-tracker update --status "进行中" --step "配置数据库"

# 添加问题
trae skill init-tracker add-issue --title "数据库连接问题" --priority "高" --assignee "张三"

# 添加建议
trae skill init-tracker add-suggestion --title "添加自动化测试" --description "建议集成 Jest 测试框架"

# 分析技术栈
trae skill init-tracker analyze-tech-stack

# 获取技术使用建议
trae skill init-tracker get-tech-recommendations

# 添加任务提醒
trae skill init-tracker add-reminder --title "API 集成" --description "使用项目已有的 Axios 库进行 API 调用" --priority "中"

# 获取任务提醒
trae skill init-tracker get-reminders
```

### CI/CD 集成

可在 CI/CD 配置文件中添加以下步骤：

```yaml
jobs:
  initialization-check:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: 检查初始化状态
        run: trae skill init-tracker check
      - name: 更新构建状态
        run: trae skill init-tracker update --status "构建中" --step "CI 构建"
```

## 数据存储

执行过程和状态信息会存储在：
- `.trae/skills/init-tracker/state.json` - 状态数据
- `.trae/skills/init-tracker/history.md` - 执行历史
- `.trae/skills/init-tracker/issues.md` - 问题清单
- `.trae/skills/init-tracker/suggestions.md` - 建议清单
- `.trae/skills/init-tracker/tech-stack.json` - 技术栈分析结果
- `.trae/skills/init-tracker/tech-recommendations.md` - 技术使用建议
- `.trae/skills/init-tracker/reminders.md` - 任务提醒清单

## 最佳实践

1. **定期更新**：在项目初始化的每个重要阶段更新状态
2. **详细记录**：记录每一步的执行命令和结果，便于后续回顾
3. **明确责任**：为每个待处理问题指定明确的责任人
4. **及时反馈**：根据实时进展调整计划和优先级
5. **持续改进**：根据执行过程中的经验，优化初始化流程
6. **技术栈分析**：在项目开始时进行全面的技术栈分析，为后续决策提供依据
7. **技术一致性**：严格遵循项目已有的技术栈，避免引入不必要的新技术
8. **插件优先**：优先使用成熟的插件，评估其兼容性和维护状态
9. **任务规划**：根据技术栈分析结果，制定合理的任务执行计划
10. **提醒管理**：定期检查和更新任务提醒，确保后续工作的顺利进行

## 示例工作流

1. **项目启动**：运行初始化检测，确认初始状态
2. **技术栈分析**：执行技术栈分析，识别项目使用的技术、UI框架和包管理器
3. **技术使用规划**：基于分析结果，制定技术使用策略，确定插件优先使用方案
4. **执行初始化**：按照计划执行初始化步骤，记录每一步
5. **问题跟踪**：发现问题时及时添加到问题清单
6. **任务提醒设置**：为后续任务添加详细的提醒和技术使用要求
7. **状态更新**：完成重要里程碑后更新状态
8. **技术验证**：验证技术栈的正确使用和插件的有效集成
9. **提醒检查**：定期检查任务提醒，确保后续工作的顺利进行
10. **总结归档**：初始化完成后，生成完整的执行报告

通过使用该技能，可以确保项目初始化过程的透明性和可追溯性，提高团队协作效率，并为后续的项目维护和扩展提供有价值的参考。
